Slide 3
System Architecture
0:50 – 1:50
  • Walk the diagram left to right, top to bottom: START board → LoRa → FINISH board → HTTPS → API Gateway.
  • At API Gateway: "a Lambda function stores the run in DynamoDB and publishes to an SNS topic."
  • From SNS: "a second Lambda picks it up and pushes it to every connected browser via WebSocket."
  • At the bottom: "the browser dashboard gets a live update in under 500ms."
  • If anyone asks about the two boards: there's a START board at the start line and a FINISH board at the finish — they talk over LoRa radio.
This is the most important slide. Don't rush it. If you can, point at each box as you say its name — helps on a screen share.
  • SNS is the fan-out. Right now it only has the one subscriber, but it would be trivial to add logging, SMS alerts, etc.
  • DynamoDB schema: timing-runs table (partition: date, sort: timestamp).
Slide 4
Hardware
1:50 – 2:40
  • "Heltec LoRa 32 V3" — ESP32 chip with an SX1262 LoRa radio and a tiny OLED display, all on one $25 board.
  • LoRa is a long-range radio protocol. No WiFi needed at the track — the two boards talk directly at 905 MHz, up to 500m outdoors.
  • Only the FINISH board needs WiFi — to reach AWS. The radio link is purely board-to-board.
  • Firmware is C++ (Arduino framework). Interrupt-driven LoRa receive — no polling, just a flag that fires when a packet arrives.
  • FINISH board supports up to 10 concurrent runs in a queue (FIFO) — so you could stage multiple cars.
If you have the boards on your desk: hold them up briefly for the camera. Two second moment — much more real than a slide.
Slide 5
Cloud Stack
2:40 – 3:30
  • Four Lambda functions — Python 3.12. Two handle REST (store timing data), two handle WebSocket lifecycle (connect/disconnect).
  • DynamoDB for run history, SNS for fan-out to the WebSocket broadcaster.
  • Terraform — all infrastructure is code. terraform apply re-zips and deploys Lambdas automatically. Mention this specifically — the PO will notice.
  • WebSocket keeps a DynamoDB table of active connectionIds. On each timing event, ws-broadcast iterates and pushes to every connected browser.
  • Dashboard: elikloft.com/timing — unlimited concurrent viewers.
Lead with Terraform — "all of this is infrastructure as code, version-controlled, no manual console clicks." That lands well with the unix crowd.
Don't get into IAM policy details unless asked. Stay at the "what and why" level.

Takeaways
4:30 – 5:00
